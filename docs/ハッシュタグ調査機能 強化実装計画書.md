## 1. 改善の目的と方針

### 現状の課題
「ハッシュタグ調査」タブは現在、**設定したシードキーワードの固定的な統計表示**に留まっている。
表示内容（関連タグ・投稿件数）は毎バッチ更新されるが、**前回との差分・変化の方向性**が見えない。
また、すでにシステムが収集している `note_ranking`（人気記事のタグ×スキ数）が活用されていない。

### 改善方針
以下の2つのデータソースをシステム内部から追加費用ゼロで活用する。

| データソース | 活用アプローチ | 回答できる問い |
|:---|:---|:---|
| `note_ranking.tags × like_count` | **共起タグ分析**（毎バッチ再計算） | 「今、人気記事の中でこのタグと一緒に使われているキーワードは何か」 |
| `note_hashtags.related_tags` の履歴 | **変化ログ**（新テーブルに蓄積） | 「noteが認識する関連タグが前回から増えたか・消えたか」 |

この2軸を組み合わせることで、**周辺キーワードの浮上・消滅をリアルタイム近似**で把握できる。

## 2. データ層の変更

### 2-1. 新テーブル: `note_hashtag_history`

バッチ実行ごとに `note_hashtags` のスナップショットを保存する。

```sql
CREATE TABLE IF NOT EXISTS note_hashtag_history (
    hashtag     TEXT    NOT NULL,
    post_count  INTEGER DEFAULT 0,
    related_tags TEXT,          -- カンマ区切り（当時の値）
    batch_date  TEXT    NOT NULL, -- YYYY-MM-DD
    PRIMARY KEY (hashtag, batch_date)
)
```

**既存テーブルへの変更**: なし（`note_hashtags` は現行のまま「最新値のキャッシュ」として継続）

### 2-2. `db.py` への追加関数

```python
def insert_note_hashtag_history(hashtag: str, post_count: int, related_tags: list, batch_date: str):
    """バッチ実行ごとのハッシュタグスナップショットを保存する。"""
    conn = get_connection()
    cursor = conn.cursor()
    cursor.execute("""
        INSERT OR REPLACE INTO note_hashtag_history (hashtag, post_count, related_tags, batch_date)
        VALUES (?, ?, ?, ?)
    """, (hashtag, post_count, ",".join(related_tags), batch_date))
    conn.commit()
    conn.close()

def get_hashtag_history(hashtag: str, limit: int = 10) -> list:
    """指定ハッシュタグの投稿件数推移を返す（新しい順）。"""
    conn = get_connection()
    cursor = conn.cursor()
    cursor.execute("""
        SELECT batch_date, post_count, related_tags
        FROM note_hashtag_history
        WHERE hashtag = ?
        ORDER BY batch_date DESC LIMIT ?
    """, (hashtag, limit))
    rows = cursor.fetchall()
    conn.close()
    result = []
    for row in rows:
        d = dict(row)
        d["related_tags"] = d["related_tags"].split(",") if d["related_tags"] else []
        result.append(d)
    return result

def get_related_tags_diff(hashtag: str) -> dict:
    """直近2回のバッチ間での related_tags の差分を返す。"""
    history = get_hashtag_history(hashtag, limit=2)
    if len(history) < 2:
        return {"new": [], "removed": [], "stable": history[0]["related_tags"] if history else []}
    current_tags = set(history[0]["related_tags"])
    prev_tags    = set(history[1]["related_tags"])
    return {
        "new":     sorted(current_tags - prev_tags),
        "removed": sorted(prev_tags - current_tags),
        "stable":  sorted(current_tags & prev_tags),
        "post_count_delta": history[0]["post_count"] - history[1]["post_count"],
        "post_count_delta_pct": round(
            (history[0]["post_count"] - history[1]["post_count"])
            / max(history[1]["post_count"], 1) * 100, 1
        )
    }

def get_cooccurrence_tags(seed: str, top_n: int = 20) -> list:
    """note_ranking から、シードタグと共起するタグを like_count 重み付きで返す。"""
    conn = get_connection()
    cursor = conn.cursor()
    cursor.execute("SELECT tags, like_count FROM note_ranking")
    rows = cursor.fetchall()
    conn.close()

    from collections import defaultdict
    weights: dict = defaultdict(float)
    counts:  dict = defaultdict(int)

    for row in rows:
        tags = [t.strip() for t in (row["tags"] or "").split(",") if t.strip()]
        lower_tags = [t.lower() for t in tags]
        if seed.lower() not in lower_tags:
            continue
        likes = max(row["like_count"] or 0, 1)
        for tag in tags:
            if tag.lower() != seed.lower():
                weights[tag] += likes
                counts[tag]  += 1

    results = sorted(
        [{"tag": t, "weighted_demand": round(w), "article_count": counts[t]}
         for t, w in weights.items()],
        key=lambda x: x["weighted_demand"], reverse=True
    )
    return results[:top_n]
```

## 3. APIの変更（`main.py`）

### 3-1. 既存 `GET /api/note-hashtags` の拡張

クエリパラメータ `with_diff=true` を追加し、変化情報を付与して返す。

```python
@app.get("/api/note-hashtags")
async def get_note_hashtags(cross_media: bool = False, with_diff: bool = False):
    results = db.get_all_note_hashtags()
    # cross_media の処理（既存）...
    if with_diff:
        for h in results:
            h["diff"] = db.get_related_tags_diff(h["hashtag"])
    return {"status": "success", "data": results}
```

### 3-2. 新規 `GET /api/hashtag-cooccurrence`

```python
@app.get("/api/hashtag-cooccurrence")
def get_hashtag_cooccurrence(seeds: str = ""):
    """
    seeds: カンマ区切りのシードキーワード（省略時は設定から全シードを使用）
    """
    if seeds:
        seed_list = [s.strip() for s in seeds.split(",") if s.strip()]
    else:
        seed_list = db.get_settings().get("seed_keywords", [])

    result = {}
    for seed in seed_list:
        result[seed] = db.get_cooccurrence_tags(seed, top_n=20)

    return {"status": "success", "data": result}
```

### 3-3. 新規 `GET /api/hashtag-history/{hashtag}`

```python
@app.get("/api/hashtag-history/{hashtag}")
def get_hashtag_history(hashtag: str, limit: int = 10):
    history = db.get_hashtag_history(hashtag, limit=limit)
    return {"status": "success", "data": history}
```

**状態バッジの定義**:
| バッジ | 条件 | 意味 |
|:---:|:---|:---|
| 🆕 | 前回バッチに存在しなかったキーワード | 新出現 |
| ↑ | 共起需要が前回比 +20% 以上 | 急上昇 |
| → | 前回比 ±20% 以内 | 安定 |
| ↓ | 共起需要が前回比 -20% 以下 | 減退（薄く表示） |

**前回データの保存方法**: `get_cooccurrence_tags()` の結果もバッチ日付と一緒に `note_hashtag_history` と同じ考え方でキャッシュする（詳細は 5-2 節参照）。

## 5-2. 共起データの前回比較のための補足スキーマ

「浮上ワード」タブの `🆕/↑` バッジには、前回バッチの共起結果との比較が必要。
`note_hashtag_history` と同様のアプローチで、共起タグの重みスナップショットも保存する。

```sql
CREATE TABLE IF NOT EXISTS note_cooccurrence_history (
    seed        TEXT    NOT NULL,
    tag         TEXT    NOT NULL,
    weighted_demand INTEGER DEFAULT 0,
    article_count   INTEGER DEFAULT 0,
    batch_date  TEXT    NOT NULL,
    PRIMARY KEY (seed, tag, batch_date)
)
```

バッチ完了後に `get_cooccurrence_tags()` の結果をこのテーブルに書き込む（`analyzer.py` の `run_analysis_pipeline()` 末尾が適切な呼び出し箇所）。
