# トレンドキーワード自動抽出・分析システム 実装計画 (Docker & shadcn/ui & UX強化版)

要件定義・仕様書に基づき、X（Yahoo!リアルタイム検索）とnoteのトレンドデータを収集・突合・分析し、ダッシュボードで可視化するシステムを構築します。

本システムは、外部APIや巨大なローカルLLMを使用せず、ローカル環境で動作する軽量なNLPモデル（Sentence Transformers）を用いてキーワードの関連度を判定します。
フロントエンドUIには **`shadcn/ui` (React + TypeScript + Tailwind CSS)** を採用し、極めてプレミアムで洗練されたデザインと、ユーザーが直感的に操作できる管理ダッシュボードを構築します。

開発・実行環境は **Docker (Docker Compose)** を使用します。進捗管理は `docs/task.md` にて随時更新します。

## 1. 技術スタック
- **コンテナ仮想化**: `Docker`, `Docker Compose`
- **バックエンド（データ収集・処理・API）**: Python
  - ベースイメージ: `mcr.microsoft.com/playwright/python:v1.40.0-jammy` (Playwright公式イメージ)
  - データベース: `sqlite3` (設定およびバッチステータスの保存用)
  - ドメイン判定: `sentence-transformers`
  - APIサーバー: `FastAPI` (非同期バックグラウンドタスク処理)
- **フロントエンド（ダッシュボード・設定UI）**: Node.js + Vite (React + TypeScript + Tailwind CSS)
  - UIコンポーネント: `shadcn/ui` (Radix UI)
  - グラフ描画: `Recharts` (React用チャートライブラリ)

## 2. データベース設計
SQLiteに以下のテーブルを作成します。
- `x_trends` (X/Yahooトレンド履歴)
- `note_ranking` (note上位記事データ)
- `keyword_analysis` (突合・スコア結果)
- `settings` (システム設定情報) - シードワード、ターゲットテーマ、NGワード、しきい値、noteカテゴリの保存用
- `batch_status` (バッチ実行進捗ステータス) - `status` (実行状態), `message` (進捗ログ), `progress` (％), `updated_at` (タイムスタンプ)

## 4. 追加仕様（UX向上策）
1. **APIプロキシ機能**: フロントエンドからのAPIリクエスト (`/api/*`) を、Vite開発サーバー経由で直接バックエンドコンテナにプロキシし、CORS問題の解消とコードのポータビリティを担保します。
2. **バッチ処理進捗の可視化**: 収集バッチ実行中、収集フェーズごとにステータス（例: "Yahooトレンド収集中...", "NLP判定中 (10/30)..."）と進捗パーセンテージをDBに記録し、UI上のプログレスバーにリアルタイム反映します。
3. **グラフ・テーブル相互インタラクション**: 4象限グラフ上の任意の点（キーワード）をクリックすると、詳細データテーブルの該当レコードに自動でスクロール＆ハイライトします。
4. **類似度テストツール（シミュレーター）**: 設定画面にテキストを入力して「テスト」をクリックすると、現在のテーマとのコサイン類似度スコアを瞬時に計算し判定結果を返す機能を搭載し、しきい値調整を容易にします。
