# トレンドキーワード自動抽出・分析システム ユーザーマニュアル

本システムは、X（旧Twitter、Yahoo!リアルタイムトレンド）の爆発力と、note（techカテゴリ人気記事）の深層需要を突合し、ローカルNLP（意味分析）を用いて「今、技術トレンドとして本当に重要なキーワード」を自動抽出・4象限分析するシステムです。

## 1. システム概要

### 課題解決の背景
情報発信者や技術リサーチャーにとって、「今どのような技術キーワードがバズっているか（X）」「何について解説した記事が実際に読まれているか（note）」を両面から把握することは極めて重要です。しかし、世の中のトレンドには技術とは無関係なエンタメ情報や日常ニュースが多く混ざっており、これらを手動で選別するのは多大な労力がかかります。

本システムは、**ローカルNLP（Sentence Transformers）** を利用して技術無関係なトレンドを完全自動で排除し、Xの「拡散度」とnoteの「実需（スキ数）」を掛け合わせて以下の4象限に自動分類します。

*   **第1象限（スター候補）**: Xでバズっており、かつnoteでも需要（スキ数）が高い、今すぐ取り組むべき強力なトレンド。
*   **第2象限（安定需要）**: Xでの爆発力はないが、noteでは安定して読まれている（スキ数が多い）実需系キーワード。
*   **第3象限（対象外）**: Xでのバズもnote需要も低く、かつ技術的文脈から遠い（または設定したしきい値以下の）キーワード。
*   **第4象限（急上昇トレンド）**: Xで急速にバズり始めたが、まだnoteでの発信が少ない、先行者利益を狙えるキーワード。

## 2. 動作環境・セットアップ

本システムは、DockerおよびDocker Compose環境さえあれば、ローカルPCのCPUのみ（有料APIキー不要）で即座に動作します。

### 前提条件
*   **Docker / Docker Desktop**: インストール済みであること
*   **Docker Compose**: 使用可能であること

### 起動手順
1.  プロジェクトのルートディレクトリに移動します。
2.  以下のコマンドを実行してコンテナをビルド・起動します。
    ```bash
    docker compose up -d --build
    ```
    *初回起動時は、NLPモデル (`intfloat/multilingual-e5-small`) の自動ダウンロードやPlaywrightのブラウザダウンロードが走るため、数分程度かかります。*

3.  起動後、ブラウザで以下のURLを開きます。
    *   **Web UI (フロントエンド)**: `http://localhost:5173/`
    *   **APIドキュメント (FastAPIバックエンド)**: `http://localhost:8000/docs`

### 停止手順
動作を終了する場合は、以下のコマンドを実行します。
```bash
docker compose down
```

## 3. 画面構成と基本操作

### ダッシュボード (Dashboard)

ダッシュボードは、収集したキーワードの分析結果をグラフィカルに可視化し、詳細を追うためのメイン画面です。

#### 1. データの新規収集実行
*   **「データ収集・分析を実行」ボタン**:
    クリックすると、非同期でYahoo!リアルタイムトレンドのスクレイピングと、note人気記事のAPI収集が開始されます。
    実行中は進捗状況（プログレスバー）が表示され、処理が完了すると自動でダッシュボードが更新されます。

#### 2. 4象限マトリクスグラフ (Recharts)
*   収集・突合されたキーワードが、X軸（note需要スコア）とY軸（Xバズスコア）の平面上にプロットされます。
*   プロットされた点にマウスカーソルを合わせる（ホバーする）と、キーワード名、詳細なスコア、分類（象限）がツールチップで表示されます。
*   グラフ上の点をクリックすると、下部の「キーワード詳細テーブル」の該当行へスクロールし、ハイライト表示されます。

#### 3. キーワード詳細テーブル
*   収集されたキーワードの一覧をテーブル形式で表示します。
*   検索バーによるキーワード絞り込みや、各カラム（Xスコア、noteスコア、分類ステータスなど）でのソートが可能です。

---

### システム設定 (Settings)

システム設定画面では、NLPによるキーワード抽出の精度向上や、収集対象のカスタマイズを行えます。設定は保存され、次回の収集実行時から即座に適用されます。

#### 1. 基本設定
*   **ターゲットテーマ**:
    NLPの類似度判定の基準となる文章を入力します。
    *例: 「Web開発、React、Next.js、TypeScript、フロントエンド、バックエンド技術に関する話題」*
*   **類似度しきい値 (Similarity Threshold)**:
    ターゲットテーマとの類似度がこの値（デフォルト: `0.81`）未満のキーワードは「対象外」として除外され、4象限グラフ上にもプロットされません。
*   **NGワードリスト**:
    除外したいキーワードをカンマ区切りで入力します。ここに登録された語を含むトレンドは、類似度にかかわらず無条件で除外されます。
    *例: 「地震, 運行情報, 遅延, 事故」*
*   **note収集カテゴリ**:
    note APIから取得するカテゴリ（デフォルト: `tech`）を指定します。

#### 2. 類似度テスト（シミュレーター）
*   現在の「ターゲットテーマ」と「類似度しきい値」の設定で、特定のキーワードがシステムを通過（採用）するかどうかを事前テストできます。
*   テストしたいキーワードを入力して「テスト実行」をクリックすると、E5モデルを用いた類似度スコアと、採用判定結果（True / False）がリアルタイムに表示されます。

## 4. 機能と運用の詳細

### ターゲットテーマ（キーワード）の設定のコツ
本システムが採用している `intfloat/multilingual-e5-small` は文脈の意味を比較する高精度な多言語埋め込みモデルです。
単なる単語（例: `React`）ではなく、**「何に関するキーワードを抽出したいか」という具体的なコンテキスト（説明文）**をターゲットテーマに指定することが重要です。

*   **効果的な設定例**:
    > `ソフトウェア開発、AI技術、プログラミング、インフラ、Webサービス、SaaS開発に関する技術的なトレンドや話題`
*   **あまり効果的でない設定例**:
    > `技術, 開発, プログラミング` (単語のみだと判定が曖昧になり、類似度スコアのブレが大きくなります)

### 類似度しきい値のチューニング
E5モデルは類似度スコアが比較的高く出やすい（無関係な日常語でも0.75〜0.78程度になる）性質があります。
*   **おすすめのしきい値**: **`0.81`**
*   **厳しめに絞り込みたい場合**: **`0.82`〜`0.84`** に設定すると、技術的な専門用語や直球のトピックのみが残ります。
*   **広く収集したい場合**: **`0.79`〜`0.80`** に設定すると、技術に隣接するビジネスやガジェットのトレンドも拾うことができます。

*設定の際は、まず「類似度テスト」機能でいくつかのテストワード（例: 「Python」「大雨」「自動運転」）を入力し、狙い通りの判定になるか確認してください。*

## 5. トラブルシューティング

### Q1. `docker compose up` が起動途中でエラーになる、または立ち上がらない
*   **原因①: Dockerデーモンが起動していない**
    *   Docker Desktopが起動しているか確認してください。
*   **原因②: ポートの競合**
    *   すでにホストマシンで `5173` ポート（Vite）や `8000` ポート（FastAPI）が使われている可能性があります。競合しているプロセスを停止するか、`docker-compose.yml` 内の `ports` マッピングを変更してください。

### Q2. データ収集を実行すると「収集失敗」またはタイムアウトする
*   **原因①: Yahoo!の仕様変更によるスクレイピングエラー**
    *   Yahoo!リアルタイム検索のUIが大幅に変更された場合、PlaywrightのXPathによる解析が失敗することがあります。その場合は、`backend/scraper.py` のセレクタ設定を調整する必要があります。
*   **原因②: ネットワーク接続の問題**
    *   初回起動時やモデルのロード時、コンテナ内から外部API（Hugging Face等）へ接続できない場合、一時的に処理がブロックされます。プロキシやインターネット接続状態を確認してください。

### Q3. データ収集しても、すべてのキーワードが「対象外（第3象限）」になってしまう
*   **原因: 類似度しきい値が高すぎる**
    *   「システム設定」で設定した「類似度しきい値」が大きすぎる可能性があります（例: `0.90` など）。しきい値を `0.80` や `0.81` に下げて再試行してください。
