# トレンドキーワード自動抽出・分析システム 改善実装計画書 (Playwrightステルス・実用性強化版)

本システムが持つ「収集の自己拡張性の欠如」「手動NGワード辞書管理の負担」「Cloudflare等によるアクセス拒否」を、**Playwrightのステルス（Stealth）化・スクレイピング拡張**およびエンジニアリングによって解決し、実用的なマーケティング価値を最大化するための改善実装計画です。

---

## 1. 改善仕様とアーキテクチャ

### 1.1. 多角的なデータソースの統合（Qiita/Zenn/Brain/Tips）
ユーザーが設定した「シードワード」のみに依存するフローから脱却し、無料テック情報から有料コンテンツ市場までを網羅したトレンドキーワードを自動収集します。
- **Qiitaトレンドの取得**: 急上昇タグ（Qiita APIまたはデイリーフィード）から最新トレンドタグを抽出。
- **Zennトレンドの取得**: ZennのRSSフィード（`https://zenn.dev/feed`）から人気記事のキーワードを自動パース。
- **Brain（ブレイン）ランキングの取得**: テック・ビジネス系カテゴリのランキング上位コンテンツのタイトルおよびタグ情報を収集。
- **Tips（チップス）ランキングの取得**: ランキング上位のコンテンツ情報を収集。
- **マージロジック**: 取得した語句を「実需キーワード候補」にマージすることで、「無料の技術トレンド」と「有料の金銭的実需トレンド」を同時に捕捉します。

### 1.2. Playwrightステルス化によるボット対策突破（Brain/Tips/Yahoo!）
BrainやTips、Yahoo!等のプラットフォームで導入されているCloudflare等のボット検知システム（WAF）を回避し、安定したデータ収集を行います。
- **`playwright-stealth` の導入**: ヘッドレスブラウザ特有のシグネチャ（`navigator.webdriver` 等）を偽装し、ボット検知を回避。
- **動的アクセスペーシングとフィンガープリント偽装**:
  - ランダムなユーザーエージェント（User-Agent）と画面解像度の適用。
  - 人間の操作を模した動的スリープ（ミリ秒単位のゆらぎ）をリクエスト間に自動挿入。
  - Cookieやローカルストレージの適切なハンドリングによるアクセス継続性の担保。

### 1.3. 動的ネガティブテーマ自動生成
ユーザー自身がネガティブワード（NGワードリスト、日常会話、スポーツ等）を手動で登録・更新し続ける手間をゼロにします。
- **時事RSSフィードの動的取得**: システム実行時に、一般時事系（Yahoo!ニュース等）の最新トピックフィードを自動取得（最大30件）。
- **リアルタイムノイズベクトル化**: 取得した「一般ニュースの見出し」をその場で `Sentence Transformers` でエンコードし、動的な「ネガティブ埋め込みベクトル」を構成。
- **相対類似度減算への自動適用**: 判定対象キーワードと、自動生成された「当日一般トピックベクトル」とのコサイン類似度を算出し、これをネガティブスコアとして採用。
- **効果**: その日の時事ノイズ（オリンピック、地震、芸能等）を定義メンテなしで自動排除します。

### 1.4. ブルーオーシャン度（執筆優先度：Priority Score）の明示
「需要と供給の比率」から、ユーザーが「今最も書く価値があるコンテンツ」をひと目で判断できるようにします。
- **供給数（Supply Count）の動的調査**: note/Qiita等の検索APIを叩き、対象キーワードでの「既存コンテンツ総数」を取得。
- **ブルーオーシャン度の算出公式**:
  $$\text{Priority Score} = \frac{\text{Demand Score (話題性・スキ数・投稿数などの総和)}}{\log_{10}(\text{Supply Count (既存記事数)} + 2)}$$
- **効果**: 需要が高く、競合が極めて少ないキーワードを「ブルーオーシャン（第1象限の中でも最優先トピック）」として自動選別します。

## 2. データベース設計の拡張

SQLiteデータベースのテーブルスキーマを以下のように拡張します。

### Table: `keyword_analysis` (カラム追加)
- `supply_count` (INTEGER) - 既存の競合コンテンツ総数（供給量）
- `priority_score` (REAL) - ブルーオーシャン度（執筆優先度スコア）
